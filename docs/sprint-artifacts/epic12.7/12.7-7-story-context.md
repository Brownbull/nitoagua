# Story 12.7-7: Admin Responsive UI Fixes - Story Context

## Technical Analysis

### Current Implementation

**File:** `src/components/admin/provider-detail-panel.tsx`

The provider detail panel has buttons that get cut off on smaller screens.

### Root Cause

Likely missing responsive classes:
- Fixed width buttons
- No flex-wrap on button container
- Insufficient padding on smaller screens

### Files to Modify

| File | Change |
|------|--------|
| `src/components/admin/provider-detail-panel.tsx` | Add responsive classes |

### CSS Fix Pattern

```tsx
// Current (likely)
<div className="flex gap-2">
  <Button>Suspender</Button>
  <Button>Banear</Button>
  <Button>Comisión</Button>
</div>

// Fixed
<div className="flex flex-wrap gap-2">
  <Button className="flex-1 min-w-[100px]">Suspender</Button>
  <Button className="flex-1 min-w-[100px]">Banear</Button>
  <Button className="flex-1 min-w-[100px]">Comisión</Button>
</div>

// Or stack on mobile
<div className="flex flex-col sm:flex-row gap-2">
  ...
</div>
```

### Testing Approach

1. Test at 320px width (smallest mobile)
2. Test at 768px width (tablet)
3. Test at 1024px+ (desktop)
4. Verify all buttons are visible and clickable
