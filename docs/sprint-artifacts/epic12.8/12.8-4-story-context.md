# Story 12.8-4: Admin Orders Mobile UX - Context

## Quick Reference

| Field | Value |
|-------|-------|
| **Story** | 12.8-4: Admin Orders Mobile UX |
| **Priority** | P1 (High) |
| **Points** | 5 |
| **Bugs** | BUG-R2-007, BUG-R2-008 |

## Problem Summary

1. Orders page too cluttered on mobile
2. Filter dropdown text cut off
3. 6 status cards take too much space
4. Order detail header cramped

## Key Files

| File | Purpose |
|------|---------|
| `src/components/admin/orders-table.tsx` | Collapsible status cards |
| `src/app/admin/orders/page.tsx` | Compact header |
| `src/app/admin/orders/[id]/page.tsx` | Status on own row |

## Implementation Steps

1. Make status cards collapsible on mobile (< 768px)
2. Create compact header: title+count left, live+timer right
3. Fix dropdown width (max-w-[200px])
4. Put status and sort on one row, search below
5. Order detail: status on its own row

## Responsive Strategy

```tsx
// Mobile: hidden, show dropdown
<div className="md:hidden">
  <select>Status filter</select>
</div>

// Desktop: show all 6 cards
<div className="hidden md:grid grid-cols-6">
  {/* Status cards */}
</div>
```
