# UX Audit Workflow Validation Checklist

## Pre-Audit Checklist
- [ ] Mockup files are accessible and can be loaded
- [ ] Flow name is clearly defined
- [ ] User context/background is understood
- [ ] All screens in the mockup have been captured

## During Roast Checklist
- [ ] All relevant agents have provided feedback
- [ ] User clarifications have been captured for ambiguous points
- [ ] Business model questions have been addressed
- [ ] Technical feasibility concerns have been raised
- [ ] Edge cases and error states have been considered
- [ ] Accessibility concerns have been noted
- [ ] Missing screens/flows have been identified

## Audit Document Quality Checklist
- [ ] Executive summary accurately reflects findings
- [ ] All clarifications from session are documented
- [ ] Each screen has been audited with specific issues
- [ ] Issues have clear current state, problem, and recommendation
- [ ] Priority levels are assigned (P0-P3)
- [ ] New screens required are documented with purpose
- [ ] Priority matrix is complete and actionable
- [ ] Key decisions are documented
- [ ] Next steps are clear for each role (UX, Architect, PM, Dev)

## Output Quality Checklist
- [ ] Audit document is saved to correct location
- [ ] Document follows template structure
- [ ] All agent attributions are included
- [ ] Recommendations are specific and actionable
- [ ] Effort estimates are reasonable
- [ ] Document can feed into PRD/Architecture/Epic planning

## Follow-up Readiness Checklist
- [ ] User knows next workflow options
- [ ] Audit findings are ready for epic extraction
- [ ] Technical concerns are flagged for architecture review
- [ ] UX designer has clear action items
